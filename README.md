## О проекте

Service Desk - это готовый к встраиванию модуль технической поддержки, который можно интегрировать в любой существующий сайт за несколько минут. Модуль представляет собой полноценную систему учёта обращений с разграничением ролей пользователей, виджетом поддержки и панелью администратора.

### Ключевые особенности

- Лёгкое встраивание в любой сайт (одна строка кода)
- Трёхуровневая система ролей: пользователь, специалист, администратор
- Встраиваемый виджет поддержки с изолированными стилями (префикс `ws-`)
- Автономная работа - не требует изменений в существующей кодовой базе сайта
- Email-уведомления о статусе обращений и ответах
- Панель администратора с управлением пользователями
- Статистика обращений в реальном времени
- Полная история переписки по каждому обращению
- Адаптивный дизайн для всех устройств
- Безопасная сессионная аутентификация

## Демонстрационный сайт

## Быстрая интеграция на ваш сайт

Для добавления модуля поддержки на ваш сайт достаточно добавить одну строку кода перед закрывающим тегом `body`:

```html
<script src="https://your-domain.com/widget.js"></script>
```

### Что происходит после добавления кода

1. На сайте появляется плавающая кнопка в правом нижнем углу
2. Кнопка автоматически скрывается для неавторизованных пользователей
3. После авторизации пользователя кнопка становится активной
4. При нажатии открывается панель поддержки с полным функционалом

### Изоляция стилей

Все стили виджета имеют префикс `ws-`, что гарантирует отсутствие конфликтов со стилями вашего сайта. Виджет не навязывает свои стили и не ломает существующую вёрстку.

### Пример встраивания

```html
<!DOCTYPE html>
<html>
<head>
    <title>Мой сайт</title>
    <!-- Ваши стили -->
    <link rel="stylesheet" href="/css/main.css">
</head>
<body>
    <!-- Содержимое вашего сайта -->
    
    <!-- Виджет поддержки - одна строка -->
    <script src="https://support.my-site.com/widget.js"></script>
</body>
</html>
```

## Структура базы данных

### Таблица users
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PRIMARY KEY | Уникальный идентификатор |
| username | TEXT UNIQUE | Логин пользователя |
| password | TEXT | Хеш пароля (SHA-256) |
| email | TEXT UNIQUE | Email пользователя |
| role | TEXT | Роль: user, specialist, admin |
| created_at | TIMESTAMP | Дата регистрации |

### Таблица tickets
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PRIMARY KEY | Номер обращения |
| user_id | INTEGER | Владелец обращения (FK) |
| name | TEXT | Имя пользователя |
| subject | TEXT | Тема обращения |
| message | TEXT | Текст обращения |
| status | TEXT | Статус: open, in_progress, closed |
| created_at | TIMESTAMP | Дата создания |
| updated_at | TIMESTAMP | Дата последнего обновления |

### Таблица replies
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PRIMARY KEY | Идентификатор сообщения |
| ticket_id | INTEGER | Обращение (FK) |
| user_id | INTEGER | Автор сообщения (FK) |
| message | TEXT | Текст сообщения |
| is_from_support | INTEGER | От поддержки (1) или пользователя (0) |
| created_at | TIMESTAMP | Дата отправки |

## Установка модуля на сервер

### Системные требования

- Python 3.10 или выше
- pip (менеджер пакетов Python)
- Доступ к SMTP серверу (для email-уведомлений)

### Пошаговая инструкция развёртывания

1. Загрузить файлы модуля на сервер

```bash
git clone https://github.com/yourusername/service-desk.git
cd service-desk
```

2. Создать виртуальное окружение

```bash
python -m venv .venv
```

3. Активировать виртуальное окружение

```bash
# Windows
.venv\Scripts\activate

# Linux/Mac
source .venv/bin/activate
```

4. Установить зависимости

```bash
pip install -r requirements.txt
```

5. Настроить переменные окружения

Создать файл `.env` в корне модуля:

```
EMAIL=your_email@gmail.com
EMAIL_PASSWORD=your_app_password
SECRET_KEY=your_secret_key_here
```

Для Gmail необходимо использовать пароль приложения:
- Включить двухфакторную аутентификацию в аккаунте Google
- Создать пароль приложения в разделе "Безопасность"

6. Запустить модуль

```bash
python app.py
```

7. Настроить веб-сервер (для production)

Для production-окружения рекомендуется использовать gunicorn:

```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

## Интеграция с существующим сайтом

### Вариант 1: Отдельный поддомен

Разместите модуль на поддомене `support.your-site.com` и подключите виджет:

```html
<script src="https://support.your-site.com/widget.js"></script>
```

### Вариант 2: Подпапка на существующем сайте

Настройте веб-сервер для проксирования запросов. Пример для Nginx:

```nginx
location /support/ {
    proxy_pass http://localhost:5000/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

Затем подключите виджет:

```html
<script src="/support/widget.js"></script>
```

### Вариант 3: Отдельный сервер

Разместите модуль на отдельном сервере и подключите виджет по полному URL:

```html
<script src="https://service-desk-server.com/widget.js"></script>
```

## Использование системы

### Для обычного пользователя (клиент сайта)

1. Нажимает на кнопку поддержки в правом нижнем углу сайта
2. Регистрируется или входит в систему
3. Создаёт обращение через виджет
4. Получает ответы поддержки на email и в виджете
5. Может отвечать на сообщения поддержки

### Для специалиста поддержки

1. Входит в панель управления по ссылке `/admin`
2. Просматривает все обращения клиентов
3. Отвечает на обращения через интерфейс
4. Изменяет статус обращений (открыто, в работе, закрыто)
5. Отслеживает статистику обращений

### Для администратора

1. Входит в панель управления `/admin`
2. Выполняет все функции специалиста
3. Управляет пользователями (добавление, изменение ролей, удаление)
4. Контролирует работу всей системы

## Возможные проблемы и решения

### Виджет не отображается

Проверьте, что пользователь авторизован. Виджет показывается только после входа в систему.

### Конфликт стилей с основным сайтом

Виджет использует уникальные CSS классы с префиксом `ws-`, что исключает конфликты со стилями основного сайта. Если конфликт всё же возникает, проверьте, что на вашем сайте нет классов с таким же префиксом.

### CORS ошибки при встраивании

Убедитесь, что в файле `app.py` правильно настроены CORS заголовки:

```python
CORS(app, supports_credentials=True, origins=['https://your-site.com'])
```

### Email-уведомления не работают

Проверьте настройки SMTP в файле `.env`. Для Gmail требуется пароль приложения, а не обычный пароль от аккаунта.

### Конфликт портов

По умолчанию модуль использует порт 5000. Если порт занят, измените его в последней строке файла `app.py`:

```python
app.run(debug=True, host='0.0.0.0', port=5001)
```
