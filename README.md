# Service Desk - Система учёта обращений в техническую поддержку

## О проекте

Service Desk - это полнофункциональное веб-приложение для организации работы службы технической поддержки. Система позволяет пользователям создавать обращения, а специалистам и администраторам - управлять ими, отвечать на запросы и анализировать статистику.

### Ключевые особенности

- Трёхуровневая система ролей: пользователь, специалист, администратор
- Встраиваемый виджет поддержки для интеграции в любой сайт
- Email-уведомления о статусе обращений и ответах
- Панель администратора с управлением пользователями
- Статистика обращений в реальном времени
- Полная история переписки по каждому обращению
- Адаптивный дизайн для всех устройств

## Технологический стек

### Бэкенд
- Python 3.10+
- Flask 2.3.0 - веб-фреймворк
- SQLite3 - база данных
- Flask-CORS - обработка кросс-доменных запросов
- SMTP (Gmail) - отправка email-уведомлений

### Фронтенд
- HTML5
- CSS3 (адаптивная вёрстка, Flexbox/Grid)
- JavaScript (ES6+)
- Fetch API для взаимодействия с сервером

### Инструменты разработки
- Git - система контроля версий
- GitHub - хостинг репозитория
- Virtualenv - изоляция окружения Python

## Структура базы данных

### Таблица users
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PRIMARY KEY | Уникальный идентификатор |
| username | TEXT UNIQUE | Логин пользователя |
| password | TEXT | Хеш пароля (SHA-256) |
| email | TEXT UNIQUE | Email пользователя |
| role | TEXT | Роль: user, specialist, admin |
| created_at | TIMESTAMP | Дата регистрации |

### Таблица tickets
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PRIMARY KEY | Номер обращения |
| user_id | INTEGER | Владелец обращения (FK) |
| name | TEXT | Имя пользователя |
| subject | TEXT | Тема обращения |
| message | TEXT | Текст обращения |
| status | TEXT | Статус: open, in_progress, closed |
| created_at | TIMESTAMP | Дата создания |
| updated_at | TIMESTAMP | Дата последнего обновления |

### Таблица replies
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PRIMARY KEY | Идентификатор сообщения |
| ticket_id | INTEGER | Обращение (FK) |
| user_id | INTEGER | Автор сообщения (FK) |
| message | TEXT | Текст сообщения |
| is_from_support | INTEGER | От поддержки (1) или пользователя (0) |
| created_at | TIMESTAMP | Дата отправки |

## API Эндпоинты

### Аутентификация

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | /api/register | Регистрация нового пользователя |
| POST | /api/login | Вход в систему |
| POST | /api/logout | Выход из системы |
| GET | /api/user | Получение информации о текущем пользователе |

### Обращения (для пользователя)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | /api/tickets | Получение списка обращений пользователя |
| POST | /api/tickets | Создание нового обращения |
| GET | /api/tickets/{id}/replies | Получение истории переписки |
| POST | /api/tickets/{id}/replies | Добавление ответа пользователя |

### Административные (для специалиста и администратора)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | /api/admin/tickets | Получение всех обращений |
| GET | /api/admin/tickets/{id}/replies | Получение переписки по обращению |
| POST | /api/admin/tickets/{id}/reply | Ответ на обращение (от поддержки) |
| PUT | /api/admin/tickets/{id}/close | Изменение статуса обращения |
| GET | /api/admin/stats | Получение статистики |

### Управление пользователями (только для администратора)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | /api/admin/users | Список всех пользователей |
| PUT | /api/admin/users/{id}/role | Изменение роли пользователя |
| DELETE | /api/admin/users/{id} | Удаление пользователя |

## Установка и запуск

### Системные требования

- Python 3.10 или выше
- pip (менеджер пакетов Python)
- Git (опционально, для клонирования репозитория)

### Пошаговая инструкция

1. Клонировать репозиторий

```bash
git clone https://github.com/yourusername/helpdesk-system.git
cd helpdesk-system
```

2. Создать виртуальное окружение

```bash
python -m venv .venv
```

3. Активировать виртуальное окружение

Windows:
```bash
.venv\Scripts\activate
```

Linux/Mac:
```bash
source .venv/bin/activate
```

4. Установить зависимости

```bash
pip install -r requirements.txt
```

5. Настроить переменные окружения

Создать файл `.env` в корне проекта:

```
EMAIL=your_email@gmail.com
EMAIL_PASSWORD=your_app_password
SECRET_KEY=your_secret_key_here
```

Для Gmail необходимо использовать пароль приложения:
- Включить двухфакторную аутентификацию в аккаунте Google
- Создать пароль приложения в разделе "Безопасность"

6. Запустить приложение

```bash
python app.py
```

7. Открыть в браузере

```
http://localhost:5000
```

## Использование системы

### Для обычного пользователя

1. Перейдите на страницу регистрации `/login.html`
2. Зарегистрируйтесь, указав логин, email и пароль
3. После входа на главной странице появится кнопка поддержки в правом нижнем углу
4. Нажмите на кнопку, чтобы открыть виджет поддержки
5. Создайте обращение, указав имя, тему и описание проблемы
6. Отслеживайте статус обращений и отвечайте на сообщения поддержки

### Для специалиста

1. Войдите в систему с учётными данными специалиста
2. Перейдите в панель управления по ссылке `/admin`
3. Просматривайте все обращения в системе
4. Отвечайте на обращения через модальное окно
5. Изменяйте статус обращений (открыто, в работе, закрыто)
6. Отслеживайте статистику обращений

### Для администратора

1. Войдите в систему с учётными данными администратора
2. Перейдите в панель управления `/admin`
3. Выполняйте все функции специалиста
4. Управляйте пользователями: изменяйте роли, удаляйте пользователей
5. Добавляйте новых пользователей с нужными ролями

## Интеграция на сторонний сайт

Для встраивания виджета поддержки на любой сайт, добавьте следующий код перед закрывающим тегом body:

```html
<script src="http://your-server:5000/widget.js"></script>
```

Виджет автоматически:
- Добавит плавающую кнопку в правый нижний угол
- Будет отображаться только для авторизованных пользователей
- Обеспечит полный функционал поддержки

## Безопасность

- Хеширование паролей с использованием SHA-256
- Сессионная аутентификация с защитой от подделки
- Защита от SQL-инъекций через параметризованные запросы
- Разделение данных пользователей через внешние ключи
- Проверка ролей на каждом защищённом эндпоинте
- CORS-настройки для безопасного взаимодействия с клиентом

## Возможные проблемы и решения

### Ошибка подключения к базе данных

Убедитесь, что у вас есть права на запись в директории проекта. База данных `support.db` будет создана автоматически при первом запуске.

### Email-уведомления не отправляются

Проверьте настройки в файле `.env`. Для Gmail необходимо использовать пароль приложения, а не обычный пароль от аккаунта.

### Конфликт портов

По умолчанию приложение использует порт 5000. Если порт занят, измените его в последней строке файла `app.py`:

```python
app.run(debug=True, host='0.0.0.0', port=5001)  # измените порт
```
