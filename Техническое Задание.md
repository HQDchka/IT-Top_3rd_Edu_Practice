# Техническое задание 
## Разработка программы учёта заявок техподдержки (Service Desk)

### 1. Что разработать
Веб-приложение для учёта обращений пользователей в техническую поддержку.

### 2. Технологии
| Компонент | Технология |
|-----------|------------|
| Бэкенд | Python (Flask или Django) |
| Фронтенд | HTML, CSS, JavaScript |
| База данных | SQLite |
| Сервер | Встроенный (Flask)|
| Установщик | PyInstaller + веб-сервер в сборке |


### 3. Функциональные требования

#### 3.1. Регистрация заявок
- Форма с полями: тема, описание, автор (автоматически из авторизации).
- Сохранение в БД с датой создания и статусом «Новая».

#### 3.2. Статусы заявок
- Новая → В работе → Выполнена.
- Возможность менять статус (кнопкой/выпадающим списком).

#### 3.3. Журнал действий (логи)
- Фиксация: кто, когда, какое действие выполнил.
- Отображение журнала в отдельной вкладке.

#### 3.4. Статистика
- Общее количество заявок.
- Количество по статусам.
- Среднее время выполнения заявки (в часах).

#### 3.5. Роли и права доступа

| Роль | Возможности |
|------|-------------|
| Пользователь | Создание заявок, просмотр и редактирование **только своих** |
| Специалист | Просмотр **всех** заявок, изменение статуса любых |
| Администратор | Всё выше + управление пользователями (добавление/удаление/смена роли) |

### 4. Защита (программными средствами)
- Авторизация по логину/паролю.
- Пароли хранятся в **хэшированном** виде (bcrypt или PBKDF2).
- На сервере проверка прав перед каждым действием.
- Логирование неудачных попыток входа (IP, время, логин).

### 5. Измерение характеристик (ПК 4.2)
Замерить и записать в отчёт:
- Время запуска приложения (от клика до открытия страницы).
- Время загрузки списка заявок.
- Потребление оперативной памяти (через диспетчер задач или `psutil`).
- Среднее время выполнения заявки (автоматически из БД).

### 6. Модификации (ПК 4.3)
Выполнить **2 доработки** по указанию руководителя, например:
1. Добавить поле «**Приоритет**» (Низкий/Средний/Высокий) в заявку.
2. Добавить **автоматическую дату закрытия** при смене статуса на «Выполнена».

### 7. Инсталляция и настройка (ПК 4.1)
- Собрать приложение в **один .exe** (PyInstaller) вместе с веб-сервером.
- Портативный режим или установщик на выбор.
- Конфигурационный файл `config.ini` (порт БД, секретный ключ).

### 9. Задача одной строкой
> **Разработать веб-приложение на Python + HTML/CSS/JS для учёта заявок техподдержки с авторизацией, ролями, замерами и двумя доработками.**
