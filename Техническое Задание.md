# Техническое задание

## Разработка модуля учёта обращений в техническую поддержку (Service Desk)

### 1. Что разработать

Веб-модуль для учёта обращений пользователей в техническую поддержку с возможностью встраивания в любой существующий сайт в виде виджета.

### 2. Технологии

| Компонент | Технология |
|-----------|------------|
| Бэкенд | Python (Flask) |
| Фронтенд | HTML, CSS, JavaScript |
| База данных | SQLite |
| Сервер | Встроенный (Flask) |
| Виджет | JavaScript с изолированными стилями |

### 3. Функциональные требования

#### 3.1. Регистрация и авторизация

- Форма регистрации: логин, email, пароль.
- Форма входа: логин, пароль.
- Пароли хранятся в хэшированном виде (SHA-256).
- Email используется для уведомлений и не запрашивается повторно при создании обращения.

#### 3.2. Виджет поддержки

- Плавающая кнопка в правом нижнем углу экрана.
- Отображается только для авторизованных пользователей.
- При нажатии открывается панель поддержки.
- Все стили виджета имеют уникальный префикс для исключения конфликтов с основным сайтом.
- Возможность встраивания на любой сайт добавлением одной строки кода.

#### 3.3. Создание обращений (виджет)

Форма с полями:
- Имя пользователя
- Тема обращения
- Текст обращения

Сохранение в БД с датой создания и статусом «Открыто».

#### 3.4. История обращений (виджет)

- Список всех обращений пользователя.
- Просмотр полной истории переписки с поддержкой.
- Возможность отвечать на сообщения поддержки.

#### 3.5. Статусы заявок

- Открыто → В работе → Закрыто.
- Изменение статуса доступно специалистам и администраторам.

#### 3.6. Панель управления (отдельная страница /admin)

- Для специалистов: просмотр всех обращений, ответы, изменение статуса, статистика.
- Для администраторов: управление пользователями (изменение ролей, удаление).

#### 3.7. Журнал действий (логи)

- Фиксация: кто, когда, какое действие выполнил.
- Отображение журнала в отдельной вкладке (для администратора).

#### 3.8. Статистика

- Общее количество заявок.
- Количество по статусам (открытые, в работе, закрытые).
- Отображение в панели управления для специалистов и администраторов.

#### 3.9. Роли и права доступа

| Роль | Возможности |
|------|-------------|
| Пользователь | Создание обращений, просмотр и ответы на **только свои** обращения через виджет |
| Специалист | Просмотр **всех** обращений, ответы на обращения, изменение статуса, просмотр статистики |
| Администратор | Всё выше + управление пользователями (добавление, изменение ролей, удаление) |

#### 3.10. Email-уведомления

- Уведомление пользователя о создании обращения.
- Уведомление пользователя об ответе от поддержки.
- Уведомление специалистов и администраторов о новом сообщении от пользователя.

### 4. Защита (программными средствами)

- Авторизация по логину/паролю с сессионной аутентификацией.
- Пароли хранятся в хэшированном виде (SHA-256).
- На сервере проверка прав перед каждым действием (декораторы login_required, role_required).
- Защита от SQL-инъекций через параметризованные запросы.
- Разделение данных пользователей через внешние ключи.
- CORS-настройки для безопасного взаимодействия с клиентом.

### 5. Измерение характеристик (ПК 4.2)

Замерить и записать в отчёт:

| Параметр | Способ измерения |
|----------|------------------|
| Время запуска приложения | От клика до открытия главной страницы |
| Время загрузки списка обращений | Через инструменты разработчика браузера (Network) |
| Потребление оперативной памяти | Диспетчер задач или psutil |
| Среднее время выполнения заявки | Автоматически из БД (разница между created_at и updated_at при статусе closed) |

### 6. Модификации (ПК 4.3)

Выполнены 2 доработки по указанию руководителя:

1. **Встраиваемый виджет поддержки** — модуль может быть интегрирован в любой сайт добавлением одной строки кода, все стили изолированы префиксом `ws-`.
2. **Автоматические email-уведомления** — при создании обращения и при ответе поддержки пользователь получает уведомление на email.

### 7. Инсталляция и настройка (ПК 4.1)

- Приложение запускается из командной строки: `python app.py`.
- База данных SQLite создаётся автоматически при первом запуске.
- Конфигурационный файл `.env` (порт, email, пароль, секретный ключ).
- Портативный режим — все файлы находятся в одной директории.

### 8. Состав проекта

| Файл | Назначение |
|------|------------|
| app.py | Основной бэкенд (Flask-приложение) |
| requirements.txt | Зависимости Python |
| .env | Переменные окружения |
| static/style.css | Стили демо-сайта |
| static/index.html | Демо-сайт (приют для животных) |
| static/login.html | Страница входа/регистрации |
| static/admin.html | Панель администратора/специалиста |
| static/widget.js | Встраиваемый виджет поддержки |

### 9. Интеграция на сторонний сайт

Для встраивания модуля на любой сайт достаточно добавить одну строку кода:

```html
<script src="https://your-domain.com/widget.js"></script>
```

Виджет автоматически:
- Добавляет плавающую кнопку в правый нижний угол
- Отображается только для авторизованных пользователей
- Не конфликтует со стилями основного сайта (префикс ws-)
- Обеспечивает полный функционал поддержки
